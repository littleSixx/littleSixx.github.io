    # Vue.set()

    Vue实例执行渲染后，再往data里面添加数据是不会响应式的，如Vue._data.student.sex = "男"或者以vm.arr[0] = 1的方式改变数组，因为Vue监听不到，而且Vue不会为数组内的成员设置getter和setter。

    解决方法：用Vue.set()即可监听到

    Vue.set(vm._data.student, 'sex', '男')
或者
vm.$set(vm._data.student, 'sex', '男')
//vm._data.student改为vm.student也行的
    1

    4

    **==注意：==**Vue.set()不允许在vm._data或者vm.里面加数据，如：Vue.set(vm.__data, 'sex', '男')或者Vue.set(vm, 'sex', '男')

    在data里面的某个对象里才行，如：Vue.set(vm._data.student, 'sex', '男')

      • 原因：在vue中的这些方法 和 js原生的方法虽然用法一样，但实质是不一样的， ==即==：

      • vm._data.arr.push !== Array.prototype.push

    # Vuex

    专门在Vue中做集中式状态（state, 数据）管理的一个Vue插件。也是一种组件间的通信方式

    mutations有 变化的 意思

    • state: 存储数据，可以理解为饭菜
    • actions：里面存方法，当用到的某些数据需要异步请求时，方便异步操作。可以理解为：服务员
    • mutations: 是最累的那个，里面存方法，就是在这里真正修改数据的。可以理解为：后厨，真正饭菜好不好吃与后厨有关

    Vue Compunents可以理解为客人，当与后厨很熟时，可以绕过服务员(actions)，直接与后厨(mutatons)沟通

    state、actions、mutations都归store管理，他们都是对象

    }
}

    state: 0
}

//创建store

    actions,
    mutations,
    state

    # history和hash模式的区别

    # hash模式

    • 什么是hash值：#号及其后面的就是hash值
    • hash值不会包含在http请求中，即不会带给服务器
    • 若以后将带有#的路径通过第三方app分享，若app校验严格，则地址会被标记为不合法

    # 区别

    1. hash模式兼容性更好
    2. history模式会导致 使用前端路由跳转后，刷新的话会变成404
      • 这里我踩过坑呀呀呀呀呀呀！！！！！
      • 原因：因为前端路由跳转是不需要网络请求的！！而刷新是需要网络请求的，刷新后会找后端的路径和资源，后端没这个资源的话自然就变成404了！！！
      • hash模式就没有这个问题。原因：哈希模式会带有'#'号，＃后面(包含#号)的路径不算做资源，也就不会向服务器请求这些资源了。

    # history模式导致前端路由跳转后刷新会404的问题，解决方法：

    • 需要后端配合：

      1. 安装npm install connect-history-api-fallback

      2. 引入：

        const history = require('connect-history-api-fallback')
        1

      3. 使用中间件

        app.use(history())
        1

    • 也可以用nginx解决，原理：用来判断这个路径是前端路由跳转的路径 还是 后端的资源

    # setup()函数

    setup()函数与name同级

    两种返回值：

    1. 若返回一个对象，则对象中的属性、方法，在模板中均可以直接使用（重点关注）
    2. 若返回一个渲染函数，则可以自定义渲染内容（了解）

    # 添加数据和方法（setup()）

    export default{
	name: 'App',

		let name = '张三';
		function test() {
			console.log(name)
		}
		return {
			name,
			test
		}
	}
}

    # reactive也可以用来处理数组

    • 在vue2中，实现数组的响应式必须要用数组的方法，直接arr[0]=xxx是不会响应式的！！！vue2中的是用Object.defineProperty来实现的。

    • 但是在vue3中，用了reactive函数，即使是arr[0]=xxx的形式也能响应式！！！vue3的实现原理和vue2不一样，使用Proxy来实现的。例如：

      const arr = reactive(['学习', '打游戏'])

    # Vue2 响应式存在的问题

    原理：vue2用的是Object.defineProperty。

    由于直接添加和删除 defineProperty是捕获不到的，所以做不到响应式

    • 直接添加或删除对象的属性不会响应式，如：

      const obj = {}
obj.a = 1
delete obj.a
//以上直接添加或删除都不是响应式的

        Vue.set(this.obj,'a',1)

this.$delete(this.obj, 'a')

    # Proxy

    window内置的构造函数Proxy：window.Proxy

    语法：const 代理对象 = new Proxy(源对象, {})

    作用：拦截对象中任意属性的变化

    **用new Proxy()**的话，无论是直接删除和添加和修改都能检测到。

    # Reflect

    作用：对源对象的属性进行操作

    ES6新增

    • 利用Reflect 获取 对象里面的数据：

      const obj = {a: 233}
Reflect(obj, 'a')//获取
      1
      2

    • 利用Reflect 修改 对象里面的数据：

      const obj = {a: 233}
Reflect.set(obj, 'a', 666)//修改
      1
      2

    • 利用Reflect 删除 对象里面的数据：

      const obj = {a: 233}
Reflect.deleteProperty(obj, 'a')//删除

    # 将Proxy和Reflect结合

    # reactive和ref的对比

    • 从定义数据角度对比：

      • ref用来定义：基本数据类型
      • reactive用来定义：对象（或数组）类型数据
      • 备注：ref也可以用来定义对象或数组，它内部会自动通过reactive转为代理对象

    • 从原理角度对比：

      • ref通过Object.defineProperty()的get和set来实现响应式（数据劫持）
      • reactive通过用Proxy来实现响应式（数据劫持），并通过Reflect操作源对象内部的数据

    • 从使用角度对比：

      • ref定义的数据：操作数据需要.value，读取数据时模板中直接读取不需要.value

      • reactive定义的数据：操作数据与读取数据：均不需要.value

    # setup()的注意点

    • setup执行的时机
      • setup函数的执行时机在生命周期中，比beforeCreate都要早。因此setup函数中的this是undefined
    • setup的参数
      • props: 值为对象，包含：组件外部传递过来，且组件内部生命接收了的属性
      • context：上下文对象
        • attrs：值为对象，包含：组件外部传递过来，但没有在props配置中声明的属性，相当于this.$attrs
        • slots：收到的插槽内容，相当于this.$slots
        • emit：分发自定义事件的函数，相当于this.$emit

    # vue3中计算属性的写法

    • 需要引用import {computed} from 'vue'

      • 简写（没有考虑计算属性被修改的情况）ps:计算属性也是属性，也有可能被修改
      setup() {
    let person = reactive({
        firstName: '张',
        lastName: '三'
    })
    let fullName = computed(() => {
        return person.firstName + person.lastName
    })
    
    return {
        person,
        fullName
    }
}

        setup() {
    let person = reactive({
        firstName: '张',
        lastName: '三'
    })
    let fullName = computed({
        get() {
            return person.firstName + '-' + person.lastName
        },
        set(value) {
            const nameArr = value.split('-')
            person.firstName = nameArr[0]
            person.lastName = nameArr[1]
        }
    })
    
    return {
        person,
        fullName
    }
}

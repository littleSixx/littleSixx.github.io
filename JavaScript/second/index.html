    # 词法(静态)作用域和动态作用域

    词法作用域：函数的作用域在函数定义的时候就决定了（取决于函数定义的位置）

    动态作用域：函数的作用域在函数调用的时候就决定了（取决于函数的调用）

    ==闭包==就是不是“通常情况下”，如果你在一个函数中嵌入了另一个函数，并且让外部一个指针引用内部的函数，例外情况就产生了。这种函数套函数的情况下，即使父级函数的执行上下文从调用栈弹出了，子级函数仍然能够访问父级函数的作用域。

    # 运行代码前，浏览器做的“准备工作”

    • 变量、函数表达式——变量声明，默认赋值为undefined
    • this——赋值
    • 函数声明——赋值

    这三种数据的准备情况我们称之为“执行上下文”或者“执行上下文环境”

    # 执行上下文

    执行上下文EC(execution context) 执行上下文功能:代码执行之前的准备工作，确定当前环境下变量的取值。

    执行上下文目的:用来存放数据的

    # 执行上下文和作用域的理解

    执行上下文是动态的，因为函数每调用一次，都会产生一个新的执行上下文环境。因为不同的调用可能就有不同的参数。

    作用域是静态的

    （对上面的话存疑？）

    执行全局代码时，会产生一个执行上下文环境，每次调用函数都又会执行上下文环境。当函数调用完成时，这个上下文环境以及其中的数据都会被消除（当然了闭包并不会乖乖就范），处于活动状态的执行上下文环境只有一个。（存疑？？）

    作用域中变量的值是在执行过程中确定的，而作用域是在函数创建时就确定的。

    # 震惊！es6之后使用let和const不存在提升是不准确的？

    可参考：

    https://stackoverflow.com/questions/31219420/are-variables-declared-with-let-or-const-not-hoisted-in-es6 （看不懂的）

    大致意思是js中无论哪种形式声明(var, let, const, function, function*, class)都会存在提升现象，不同的是， var,function,function*的声明会在提升时进行初始化赋值为 undefined，因此访问这些变量的时候，不会报 ReferenceError 异常，而使用 let,const,class 声明的变量，被提升后不会被初始化，这些变量所处的状态被称为“temporal dead zone”，此时如果访问这些变量会抛出ReferenceError 异常，看上去就像没被提升一样。

    # js临时死区（Temporal Dead Zone）

    临时死区不是词法位置上的一个范围，而是一个时间范围

    JavaScript 引擎在扫描代码发现变量声明时，要么将他们提升到作用域顶部（遇到 var 声明），要么将声明放在 TDZ 中（遇到 let 和 const 声明）。访问 TDZ 中的变量会触发运行时错误。只有执行过变量声明语句后，变量才会从 TDZ 中移除，然后方可访问。

    在这个作用域的开始直到变量的声明之前，这个变量都是处在 “临时性死区” 当中的，这个时候引用他的话会报referenceError的错误

    # Math

    Math和其它对象不同，它不是一个构造函数，它是一个工具类

    • Math.PI可以获取Π的值
    • Math.abs(-1)可以获取一个数的绝对值，此处返回1
    • Math.ceil(1.3)可以将一个小数上取整，此处返回2
    • Math.floor(1.8)可以将一个小数下取整，此处返回1
    • Math.random()可以随机生成0~1之间的随机数，是0和1之间，不会出现0也不会出现1
    • Math.round(1.3)可以将小数四舍五入，此处返回1
    • Math.max(1, 3, 2)可以返回多个数中的最大值，也能传数组，不过要在形参前加...，如：Math.max(...arr)
    • Math.min(1, 3, 2)可以返回多个数中的最小值，也能传数组，不过要在形参前加...，如：Math.min(...arr)
    • Math.pow(x, y)可以返回x的y次幂
    • Math.sqrt(2)用于对一个数进行开方运算，此处为对2进行根号
